### 1. Draggable glitchy window + upload bar — `967c03d`

The live `[ ].edited` surface is the `FunnelDrawer` panel (reached via the corner `[ ]` pill — there is no desktop tile; the full-screen `EditedApp` terminal app is unreachable). Rebuilt that panel from a fixed right-anchored side drawer into a floating, draggable window.

**New — `frontend/src/components/terminal/apps/EditedApp/EditedWindow.tsx`:**
- **Draggable** by the title bar via **Pointer Events** (`pointerdown/move/up` + `setPointerCapture`) so mouse and touchscreen Chromebooks behave identically. `touch-action: none` on the handle so touch-drag doesn't scroll the page.
- **Viewport-clamped** on every move + re-clamped on resize/rotate.
- **Position persists** across open/close via a module-level `savedPos` (the drawer unmounts off-terminal, so component state alone would reset).
- **No backdrop** — the full-screen wrapper is `pointer-events-none`; only the window box is `pointer-events-auto`. It floats over the live Party work (peek behind it = the "insider toggle" feel). Closes via the title-bar `[✕]` or auto-closes on leaving the terminal.
- **Rough / suspicious chrome** (within the dead-internet doctrine): square corners, dashed hairline border, `⠿` grip handle, `unsigned build · source unknown` subtitle, **fake-broken `[–]`/`[▢]` controls** (only `[✕]` works).
- **Upload bar at the bottom** — a thin rose progress bar that climbs → stalls near the top → resets → retries but **never completes** (a covert transmission that can't be trusted to hold). Throttled `setInterval` (~320ms; no RAF, per the ambient-ticker rule). The "N packets queued for — F" line reads `spyStore.resolved` (funnelled-intel count).
- Renders the existing `FreyChannel` (shared with the unreachable full app).

**`frontend/src/index.css`** — appended `.edited-window` glitch/chrome: rare whole-window stutter (`edited-window-jitter`), a rose tear line that rips across (`::after`), title chromatic-split (`.edited-glitch-text`), baked scanlines, flickering upload fill (`.edited-upload-fill`), and a glitchy corner launcher (`.edited-pill`). **`frontend/src/components/spy/FunnelDrawer.tsx`** — renders `EditedWindow` in place of the old side drawer.

### 2. The corner `[ ]` launcher itself draggable — `e536c78`

**Reported bug:** the window was draggable, but the corner `[ ]` pill the user actually sees and grabs was still a fixed click-to-open button — so "dragging the app" did nothing. A requirements miss (I'd made the *window* draggable, not the *launcher*).

**Fix (`FunnelDrawer.tsx`, new `FunnelPill` component):** the pill is now **drag-to-move** — Pointer Events, viewport-clamped, position persists (module-level `savedPillPos`). Tap-vs-drag is a **< 4px movement threshold**: a clean tap still opens the channel; a real drag sets `suppressClick` so the trailing `click` doesn't open it. `onClick` is kept for keyboard (Enter/Space) activation. Mouse + touch.
