## Part 0 — Word Pool icon (carryover, `13a0853` + `8d78cdb`)

The uploaded icon was byte-identical to the deployed one; the real issue
was **format**: a 1254×1254 opaque square vs the other tiles' 1536×1024
transparent landscape. Reformatted the asset to the canonical format
(flood-filled the uniform background to transparency, centered the tile
to match peers' footprint) and **deleted the special-case `h-[160px]`
render branch** — Word Pool now goes through the identical full-image
tile path. **Supersedes** the 05-25 "square + `h-[160px]`" approach
documented in the 05-25 changelog/daily.

Then it rendered too large / boxed because the browser was serving the
**old cached image** at the stable `/images/` path (serve.json caches it
1 day). Fix: **renamed** `inscription-pool-icon.png` →
`inscription-pool-icon-v2.png` to cache-bust. New rule: swapping a
`public/` image in place won't update clients — rename it.

## Part 2 — Live Open Pool matchmaking (`76292cf`) — the big one

"Open Pool" now matches real online classmates into a live race.

**Architecture: per-player drills linked by a shared `lobbyId`** (reused
the dormant scaffolding — `lobbyId` column + `inscription:lobby:*` rooms
were built for exactly this). Kept every per-pair endpoint unchanged.

- `backend/src/socket/inscriptionMatchmaking.ts` (NEW): in-memory queue
  keyed `classId:lane:weekNumber`. Pool forms on cap (`POOL_MAX_PLAYERS=5`)
  or after the wait window. Same-lane only. Ghost top-up to
  `POOL_TARGET_DESKS=4`; a lone player after the wait = solo-with-ghosts
  (still `open`). Queue is ephemeral (process memory).
- Socket protocol: `join-queue` / `leave-queue` (+ disconnect cleanup) →
  `queue-update` / `pool-formed` / `participant-progress` (keyed by
  `pairId`, so each client maps it to the right opponent desk) /
  `queue-error`.
- Per-player drills share `lobbyId` + one `wordQueue` + a synchronized
  `startAt`. Completion ranks the pool from sibling drills' real progress
  + snapshots it (refresh-safe results). Open mode is **exempt from the
  5-min cooldown** (Solo keeps cooldown + daily cap).
- Schema: `InscriptionRecording.pairId String?` + migration
  `20260526120000_add_recording_pair_id`.
- Frontend: `WaitingRoom.tsx` (NEW), store `queue` screen + matchmaking
  actions, synchronized countdown overlay in `InscriptionDrill`
  (input blocked until `startAt`), live opponent desks driven by socket
  (ghost ticker kept off them).

**v1 limitations (acceptable):** reconnect mid-race doesn't rejoin the
lobby feed; in-race drop-out freezes that desk (no "left" badge);
near-simultaneous finishes can rank opponents at slightly different
instants across clients.

## Part 3 — Prompt polish + 20s countdown (`1caa010`)

- **Word-break fix:** the per-character prompt grouped each char as its
  own flex item, so the line broke between ANY two chars (e.g. "sec"/
  "tor"). Now chars are grouped into word-units (`inline-flex`, nowrap);
  only the gaps between words are wrap points.
- **Contrast:** framed the prompt in a bordered panel + brightened
  untyped chars from `phosphor-text-faint` → standard `phosphor-text`.
- **Countdown:** `POOL_WAIT_MS` 12s → **20s**; the matchmaker sends a
  shared `formsAt_ms` deadline in `queue-update`; the waiting room shows
  a synchronized "starts in Ns" countdown.

## Part 4 — Terminal app frame fills full height (`7584a13`)

`TerminalAppFrame` wrapped children in a `height:auto` `<div className=
"relative">`, so full-height apps (Word Pool's `crt-phosphor-monitor
h-full`) resolved `h-full` against an auto-height parent and only grew to
content height — letting the cyan `crt-monitor-screen` desktop bg peek
through at the bottom. Made the wrapper `flex-1` in a flex column → it
has a definite height → `h-full` fills the frame. One fix covers every
Word Pool screen. Apps that don't fill still sit on the cyan as before.
